UK KYB Request Payload — The Address Split Strategy

Critical design decision: Worth AI sends a street-only address to Trulioo (no suite/apartment) so the Comprehensive View shows green on Address Line 1. Including a suite number causes false "nomatch" because Companies House only stores the street.

Internal storage payload

// Full address WITH suite (stored in DB):
businessPayload = {
  companyName: "Acme Limited",
  companyCountryIncorporation: "GB",
  companyStateAddress: "England",
  companyCity: "London",
  companyZip: "W1U 6RG",
  companyregno: "12345678",        // Companies House no.
  companyAddressFull: "123 Baker Street, Suite 2"
}

Sent to Trulioo (street only)

// Street ONLY (no suite) sent to Trulioo:
truliooPayload = {
  ...businessPayload,
  companyAddressFull: "123 Baker Street"
  // ↑ Suite stripped intentionally
  // → Trulioo Comprehensive View compares
  //   "123 Baker Street" vs Companies House
  //   → Address Line 1: match ✓
}

// businessPayload.companyAddressFull (full)
// stored in business_entity_address_source
// for badge comparison with submitted address

PEP Screening — UK-Specific

Who is a UK PEP?

  • Current/former senior politicians: MPs, Lords, Devolved Assembly Members
  • Government ministers and senior civil servants
  • Senior judicial officers (High Court judges+)
  • Ambassadors and high commissioners
  • Senior executives of state-owned enterprises
  • Members of international org governing bodies (UN, NATO…)
  • Immediate family and known close associates of any of the above
Basis: UK AML regulations (JMLSG guidance, Money Laundering Regulations 2017)

Where PEP hits appear

// In WatchlistHitDetails:
"PEP_results": [
  {
    "score": 1,
    "subjectMatched": "Jane Smith",
    "sourceListType": "PEP Database",
    "sourceRegion": "Europe"
  }
]

// createWatchlistHit() transforms to:
{
  listType: "PEP",
  listName: "PEP Database",
  confidence: 1.0,
  matchDetails: "Jane Smith",
  sourceRegion: "Europe"
}
PEPs are screened for BOTH the business entity (KYB) and each UBO/Director (PSC flow). Both appear as separate entries in the consolidated watchlist fact — business hits with entity_type="business", person hits with entity_type="person".

OpenAI scoring pipeline

For each AM_results hit: 1. Get title (from listName or URL slug) 2. scoreAdverseMedia(title, entityNames) via OpenAI: → keywordsScore (0–1): fraud/crime keywords → negativeSentimentScore (0–1): sentiment → entityFocusScore (0–1): article about this entity? → finalScore (0–1): weighted combination → riskLevel: "LOW" | "MEDIUM" | "HIGH" → description: human-readable risk explanation 3. Deduplicate by (link, mediaType) 4. insertAdverseMedia(): → adverse_media (business_id, counts) → adverse_media_articles (per article, all scores) Stored example: { title: "Acme Ltd fraud investigation", source: "bbc.co.uk", final_score: 0.91, risk_level: "HIGH" }

PSC (UBO/Director) Screening — Automatic UK Flow

For UK businesses: PSC screening is AUTOMATIC. No separate toggle. No manual trigger. It runs immediately after KYB verification completes with status="approved". This follows the Middesk pattern: one API call handles both business and person verification.

Decision logic

shouldScreenPSCsForBusiness("GB"): isUS = false // GB is not US internationalKYBEnabled = true // from data_integration_settings → { shouldScreen: true } // automatic for non-US // US businesses need Advanced Watchlists toggle: shouldScreenPSCsForBusiness("US"): isUS = true advancedWatchlistsEnabled = false // if toggle off → { shouldScreen: false } // Only screens US owners if toggle explicitly enabled

Person sources aggregated

  • businessData.ubos[] — PSCs from Companies House
  • businessData.directors[] — Directors from filing
  • data_owners (case-service) — applicant-submitted owners
  • Deduplication: by fullName (case-insensitive)

Atomic lock — prevents duplicate screening

// Only ONE process triggers PSC per person:
INSERT INTO integration_data.business_entity_people
  (business_entity_verification_id, name, ...)
ON CONFLICT (business_entity_verification_id, name)
DO IGNORE  ← atomic lock

// If INSERT succeeds → screen this person
// If conflict → already being screened → skip

PSC result storage

// Person hits stored with entity_type="person":
WatchlistValueMetadatum {
  type: "pep",
  entity_type: "person",    // ← distinguishes from KYB
  metadata: {
    entity_name: "John Smith",
    title: "PEP Database"
  }
}
// vs business hits: entity_type = "business"

Summary — UK Screening in One Slide

What Trulioo does for UK businesses

  • Queries Companies House for registration, directors, UBOs, addresses
  • Verifies submitted address via Comprehensive View (match/nomatch per field)
  • Screens against 20+ global watchlists simultaneously: OFAC, BIS, UN, EU, UK OFSI, UK Insolvency registers, UK Disqualified Directors, PEP databases, Adverse Media
  • Returns relevancy score per hit (1.0 = exact)
  • Automatically triggers PSC screening for all UBOs/Directors (no toggle needed)

What Worth AI does with that data

  • Stores raw response in request_response (audit trail)
  • Extracts watchlist hits with Priority 1/2 logic
  • Scores adverse media articles with OpenAI (keywords, sentiment, entity focus)
  • Consolidates business + PSC hits into watchlist computed fact
  • Surfaces source.confidence in API (entity match quality)
  • Does NOT surface watchlist hit relevancy score (0.0–1.0) — known gap
  • Builds 360 Report watchlist section from watchlist + screened_people facts
