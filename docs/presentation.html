Part 1

The Worth AI Pipeline — How It Actually Works Today

Why Industry Classification Matters

💳 Payment Compliance
MCC drives interchange rates, BSA/AML screening, and corporate card spending limits. Wrong MCC = compliance exposure.
⚖️ Risk Underwriting
Industry code is the primary denominator for default rate, fraud rate, and credit eligibility. Some lenders never touch cannabis, firearms, or crypto.
🏛️ Regulatory Reporting
UK FCA, EU EBA, and FinCEN require sector-specific classification. NAICS doesn't satisfy Companies House — UK SIC 2007 does.
🔍 KYB & AML
Industry type is a primary AML/CTF signal. A "Holding Company" registered + "Electronics Wholesale" web presence = high-risk discrepancy.

The 4 Critical Gaps — UK Business Gets NULL

Gap 1 — Resolver loop filters to US only
OpenCorporates returns "uk_sic-62012" but the resolver loop only passes entries where codeName.includes("us_naics"). UK SIC is parsed, then discarded.
Gap 2 — Trulioo .sicCode never read
Trulioo returns { naicsCode: "…", sicCode: "62012" } in the same object. The fact engine reads ?.naicsCode. .sicCode is adjacent but never accessed — one missing line of code.
Gap 3 — classification_codes has no consumer
The classification_codes fact correctly captures all jurisdiction codes (uk_sic, ca_naics…) as a Record. But no report, scorer, or Kafka handler ever reads it. It is dead code.
Gap 4 — No persistence layer for UK SIC
Even if the code were extracted, there is no core_uk_sic_code table, no uk_sic_id column on data_businesses, and no migration that adds them. The data has nowhere to go.
Result: Every UK business gets naics_id = NULL or an inappropriate US NAICS code. The UK SIC data sits in integration_data.request_response — already there, just never used.

How LSEG / Refinitiv Does It

Refinitiv Business Classification (TRBC)
Proprietary 5-level taxonomy with 10,000+ codes. Automatically cross-walked to NAICS, NACE, UK SIC, ISIC. A UK restaurant gets UK SIC 56101 as primary output, NAICS 722511 as secondary — preserving the "Licensed" status that NAICS merges away.
Semantic Discrepancy Flagging
If a company is registered as "Investment Holding" (low risk) but web presence shows "Electronics Wholesaling" (high risk for dual-use goods), the system triggers a Semantic Discrepancy Flag — not just a classification mismatch.
AML / Structure Change Signal
Frequent changes in industry codes within 12 months are treated as "Structure Change" risk — indicative of money laundering or U-Turn fraud. The classification history IS the risk signal.
Entity Resolution at Scale
Cross-references GLEIF LEI, company registry filings, beneficial ownership data, and web presence to build a 360° entity profile before classifying. Classification is a derived attribute, not the primary one.
"Top-tier providers no longer treat NAICS and UK SIC as separate buckets. They use a unified vector space where every code maps to a semantic cluster — a UK SIC 56101 is instantly comparable to a US NAICS 722511."

The Unified Global Ontology (UGO)

A FAISS vector index built from all 2,330 taxonomy codes across 6 classification systems. Each code is embedded using sentence-transformers and stored in a shared high-dimensional space.

This enables Cross-Ontology Embedding Alignment: searching "licensed restaurant" returns UK SIC 56101 as the top hit, with NAICS 722511 and ISIC 5610 as semantic neighbours — without hardcoded crosswalk tables.
